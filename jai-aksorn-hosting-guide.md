# ใจอักษร — คู่มือ Hosting / Deploy ขึ้นใช้งานจริง

ระบบที่ทำมาทั้งหมด (เว็บอ่านนิยาย + หน้าแอดมิน + ฐานข้อมูล/auth/storage บน Supabase)
ไม่มีส่วนไหนต้องใช้ "เซิร์ฟเวอร์ Node.js ของเราเอง" เลย เพราะ Supabase ทำหน้าที่
แบ็กเอนด์ให้ครบแล้ว สิ่งที่ต้อง host จริง ๆ มีแค่ **ไฟล์หน้าเว็บ (static files)**
เท่านั้น — ทำให้ deploy ง่ายและถูกกว่าระบบที่มีเซิร์ฟเวอร์เต็มรูปแบบมาก

---

## ภาพรวมสถาปัตยกรรม

```
ผู้อ่าน/นักเขียน ─┐
                 ├─→  เว็บหน้าเว็บ (index.html)  ──┐
แอดมิน ──────────┘                                ├─→  Supabase
                     หน้าแอดมิน (admin.html)  ──────┘   (Database + Auth + Storage)
```

ทั้งสองหน้าเป็นไฟล์ static (HTML/CSS/JS) เรียก Supabase ตรงจากเบราว์เซอร์ผู้ใช้
ไม่มี "เซิร์ฟเวอร์กลาง" ของเราคั่นอยู่ระหว่างทาง

---

## ขั้นตอนที่ 1 — เตรียมโค้ดใน Git repository

1. สร้าง repo บน GitHub (ฟรี) เช่น `jai-aksorn-web`
2. ใส่ไฟล์ทั้งหมดที่ทำไว้:
   - `index.html` (เว็บหลัก)
   - `admin.html`
   - `jai-aksorn-api-client.js`
   - `jai-aksorn-auth.js`
   - `jai-aksorn-storage.js`
3. Push ขึ้น GitHub

การใช้ Git ตั้งแต่ต้นสำคัญเพราะ hosting ที่แนะนำด้านล่างทำงานแบบ **"push แล้ว
ขึ้นเว็บอัตโนมัติ"** — ไม่ต้องอัปโหลดไฟล์เองทุกครั้งที่แก้ไข

---

## ขั้นตอนที่ 2 — เลือกที่ Host หน้าเว็บ (frontend)

ตัวเลือกที่เหมาะกับเว็บ static ล้วน ๆ แบบนี้ (เรียงตามความง่ายสำหรับมือใหม่):

| ตัวเลือก | จุดเด่น | ค่าใช้จ่ายเริ่มต้น |
|---|---|---|
| **Vercel** (แนะนำ) | เชื่อม GitHub แล้ว deploy อัตโนมัติทุกครั้งที่ push, HTTPS ให้ฟรี, ตั้งค่าโดเมนง่าย | ฟรี |
| **Netlify** | ใกล้เคียง Vercel มาก มี form handling ในตัวเผื่อใช้ทีหลัง | ฟรี |
| **Cloudflare Pages** | เร็วมาก มี CDN ทั่วโลกในตัว | ฟรี |

**ขั้นตอนบน Vercel (ตัวอย่าง):**
1. สมัคร vercel.com ด้วยบัญชี GitHub
2. กด "New Project" → เลือก repo `jai-aksorn-web`
3. Framework Preset เลือก "Other" (เพราะเป็น static HTML ล้วน ไม่มี build step)
4. กด Deploy — ได้ลิงก์ทดสอบทันที เช่น `jai-aksorn-web.vercel.app`
5. ทุกครั้งที่ `git push` โค้ดใหม่ เว็บจะอัปเดตอัตโนมัติภายในไม่กี่วินาที

---

## ขั้นตอนที่ 3 — ตั้งค่า Supabase ฝั่ง Production

เมื่อมีโดเมนจริงแล้ว ต้องย้อนกลับไปปรับ Supabase ให้รู้จักโดเมนนั้น:

1. **Authentication → URL Configuration**
   - Site URL: `https://jaiaksorn.com` (โดเมนจริงที่จะใช้)
   - Redirect URLs: เพิ่ม `https://jaiaksorn.com/**` (ลิงก์ยืนยันอีเมล/รีเซ็ตรหัสผ่านจะใช้ค่านี้)
   - ถ้ายังไม่ลบ URL ทดสอบ (เช่น `localhost:3000`) ออก คนร้ายอาจสวมรอยใช้ลิงก์ปลอมได้

2. **Authentication → Email Templates / SMTP**
   - ค่าเริ่มต้น Supabase ส่งอีเมลยืนยัน/OTP ให้ฟรีแต่จำกัดจำนวนต่อชั่วโมง (พอสำหรับทดสอบ)
   - เมื่อมีผู้ใช้จริงจำนวนมาก ควรต่อ SMTP ของตัวเอง (เช่น Resend, SendGrid) ที่เมนูนี้
     เพื่อให้อีเมลส่งได้ไม่จำกัดและไม่ตกไปโฟลเดอร์สแปม

3. **Project Settings → API**
   - คัดลอก `Project URL` และ `anon public key` ไปใส่ใน `jai-aksorn-api-client.js`
     (ค่านี้ปลอดภัยที่จะฝังในโค้ดฝั่งหน้าเว็บ เพราะ RLS เป็นคนป้องกันจริง ไม่ใช่กุญแจนี้)
   - **ห้ามใช้ `service_role key`** ในไฟล์ฝั่งหน้าเว็บเด็ดขาด กุญแจนั้นข้าม RLS ได้ทั้งหมด

4. **Database → Backups**
   - แพ็กเกจ Pro ของ Supabase (ประมาณ $25/เดือน) มี daily backup อัตโนมัติ —
     ควรอัปเกรดตอนเริ่มมีข้อมูลจริง/รายได้จริงเข้ามา ไม่ใช่ปล่อยไว้ที่แพ็กเกจฟรี
     ตลอดไป เพราะแพ็กเกจฟรีไม่มี backup ให้

---

## ขั้นตอนที่ 4 — โดเมนของตัวเอง

1. ซื้อโดเมน (เช่นผ่าน Cloudflare Registrar, Namecheap) เช่น `jaiaksorn.com`
2. ใน Vercel → Project Settings → Domains → ใส่โดเมน แล้วตั้งค่า DNS ตามที่ Vercel บอก
   (ปกติคือเพิ่ม CNAME record ที่ผู้ให้บริการโดเมน)
3. HTTPS จะถูกจัดการให้อัตโนมัติ (Let's Encrypt) ไม่ต้องซื้อ SSL certificate เอง
4. อย่าลืมย้อนกลับไปอัปเดต Redirect URLs ในขั้นตอนที่ 3 ให้ตรงกับโดเมนใหม่

---

## ขั้นตอนที่ 5 — แยกสภาพแวดล้อม Staging vs Production

เมื่อเริ่มมีผู้ใช้จริง แนะนำให้มี Supabase project แยก 2 ชุด:

- **Staging** — สำหรับทดสอบฟีเจอร์ใหม่ก่อนปล่อยจริง ใช้ข้อมูลปลอม
- **Production** — ข้อมูลผู้ใช้จริง เหรียญจริง สลิปจริง

วิธีทำ: สร้าง Supabase project ที่ 2 รัน schema เดียวกัน (ไฟล์ SQL ที่ทำไว้ในข้อ 1)
แล้วสลับค่า URL/key ใน `jai-aksorn-api-client.js` ตามสภาพแวดล้อมที่ deploy
(Vercel รองรับตั้งค่านี้แยกต่อ branch ได้ในเมนู Environment Variables)

## เช็กลิสต์ก่อนเปิดใช้งานจริง

- [ ] ปิด URL ทดสอบ (`localhost`) ออกจาก Redirect URLs แล้ว
- [ ] ตั้งค่า SMTP ของตัวเองแทนค่าเริ่มต้นของ Supabase (กันอีเมลไม่ส่ง/ตกสแปม)
- [ ] อัปเกรดแพ็กเกจ Supabase เป็น Pro เพื่อเปิด daily backup
- [ ] ทดสอบ flow เต็มบนโดเมนจริง: สมัคร → ยืนยันอีเมล → อ่านฟรี → โอนเงิน+แนบสลิป → แอดมินอนุมัติ → อ่านตอนที่ปลดล็อก
- [ ] ทดสอบเปิดเว็บบนมือถือจริง (ไม่ใช่แค่ย่อหน้าต่างเบราว์เซอร์)
- [ ] ตั้งรหัสผ่านบัญชีแอดมินให้แข็งแรง และอย่าล็อกอินแอดมินบนอุปกรณ์สาธารณะ

---

## ค่าใช้จ่ายโดยประมาณช่วงเริ่มต้น

| รายการ | แพ็กเกจเริ่มต้น | เมื่อเริ่มมีผู้ใช้จริง |
|---|---|---|
| Hosting หน้าเว็บ (Vercel) | ฟรี | ฟรี ถึงหลักแสน views/เดือน |
| Supabase | ฟรี (500MB DB, 1GB storage) | Pro ~$25/เดือน (แนะนำเมื่อมีรายได้จริง) |
| โดเมน | ~300-500 บาท/ปี | เท่าเดิม |
| SMTP อีเมล (Resend ฯลฯ) | ฟรี (หลักพันฉบับ/เดือน) | จ่ายตามปริมาณเมื่อเกินโควตาฟรี |

รวมแล้วช่วงเริ่มต้นแทบไม่มีค่าใช้จ่าย นอกจากค่าโดเมนปีละครั้ง — ขึ้นระบบทั้งหมด
ได้จริงโดยไม่ต้องลงทุนก้อนใหญ่ก่อน แล้วค่อยอัปเกรดตามการเติบโตจริง
